# Avaliação Técnica

## Instruções

### Introdução
Este documento descreve a avaliação técnica em termos de requisitos e resultados práticos necessários.  
A avaliação consiste em projetar e implementar um serviço requisitado.

### Requisitos
- Produzir um projeto e uma implementação, respeitando os requisitos.
- Fornecer um aplicativo totalmente funcional.

### Requisitos Funcionais
- Estão descritos no caso de uso do serviço de ordenação.
- Atenção especial à seção de "Requisitos Especiais".

### Outros Requisitos
- Clareza e facilidade de manutenção.
- Documentação clara, evitando comentários óbvios.
- Uso de elementos padrão do .NET Framework.
- Frameworks externos podem ser usados, desde que empacotados.
- Desacoplamento da interface pública.
- Aplicar padrões de design conhecidos.
- Plataforma obrigatória: .NET.

### Duração
- O tempo de implementação é livre, mas velocidade x qualidade será avaliada.

### Execução da Submissão
- Deve rodar em versão de produção da plataforma .NET.
- Entregáveis:
  - Documento com projeto da solução (UML + descrição).
  - Código fonte completo.
  - Pacote de testes implementados.

### Avaliação
- Primeira fase: execução do código.
- Segunda fase: análise da concepção e implementação.
- Exame oral para justificar decisões de projeto.
